# MOBO Digital Design System

## Thesis

MOBO Digital is a project archive for a technical digital studio. It should feel like a confident work ledger: editorial, industrial, precise, and proof-heavy. The design must not look like a generic agency template, SaaS landing page, or personal portfolio clone.

## Visual Language

- Base surface: warm archive paper, not pure white.
- Ink: near-black navy for authority.
- Accent: hazard amber for proof markers, links, and active states.
- Secondary accents: restrained clay red and data green only for categorisation.
- Typography: heavy compressed-feeling system display through scale and tracking, with clean sans body copy.
- Layout: case-file rows, oversized numbering, evidence strips, and real project screenshots.
- Imagery: use actual project screenshots. No generic illustrations, no stock photos, no artificial hero imagery.

## Composition Rules

- The homepage opens like a portfolio dossier, not a sales page.
- The work must appear above the fold through visible project evidence.
- Use large project titles, outcome labels, and client/work type metadata.
- Avoid stacked cards as the main pattern. Prefer horizontal case files, split evidence panels, and dense project indexes.
- Every project section must answer: what it was, what was built, what stack or capability it proves.

## Motion

- Hero entries rise in staggered layers.
- Case files reveal as the user scrolls.
- Project screenshots use restrained hover zoom and cursor-follow shine.
- A horizontal proof rail moves slowly to create an archive/workshop feel.

## Public/Private Boundary

- The receipt/invoice generator is an internal tool. Do not link it from the public site.
- Public navigation should only expose portfolio work, capabilities, and contact.
